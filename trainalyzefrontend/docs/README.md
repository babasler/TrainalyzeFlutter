# Trainalyze Frontend - Dokumentation

Diese Dokumentation beschreibt die Datenstrukturen, APIs und UI-Komponenten der Trainalyze Flutter App.

## 📁 Struktur

- **[entities/](./entities/)** - Datenmodelle und Entity-Klassen
- **[api/](./api/)** - API-Endpunkte und Service-Dokumentation
- **[ui/](./ui/)** - UI-Komponenten und Design-System

## 🚀 Quick Start

1. **Entity-Dokumentation**: Siehe [entities/](./entities/) für alle Datenfelder und deren Bedeutung
2. **API-Referenz**: Siehe [api/](./api/) für Backend-Integration
3. **UI-Guidelines**: Siehe [ui/](./ui/) für Design-Prinzipien

## 📝 Conventions

- **Sprache**: Deutsche Feldnamen und Kommentare
- **Datum**: ISO 8601 Format (UTC)
- **IDs**: Numerisch, auto-increment
- **Pflichtfelder**: Mit `required` markiert
- **Optionale Felder**: Mit `?` markiert
